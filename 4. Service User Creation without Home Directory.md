In response to the latest tool implementation at `xFusionCorp Industries`, the system admins require the creation of a service user account. Here are the specifics:  

Create a user named `kareem` in `App Server 1` without a home directory.

> Note: You can find the infrastructure details by clicking on the **Details of all Users and Servers** button on the top-right section of the page.

### Answer

1. SSH into Application Server 1.

```
ssh tony@stapp01
```

2. Verify the user is not already created and if not create user with `useradd`.

```
# check if user exist
cat /etc/passwd | grep kareem

# Create user
sudo useradd -M kareem
```

- `-M` option ensures that no home directory is created.

> **Note**: 
> When using `useradd` without any options to create a user, this user generally has no home directory created. The `-m` option has to be added to create a home directory. 
> 
> However, in the `/etc/login.defs`, the option can be set to always create a home directory for new users on a system wide level. To ensure that no home directory is created no matter what the defaults are we use the `-M`. 
> 
> For this exercise, looking into the `/etc/login.defs` configuration, the create home directory by default is set to yes.

```
# If useradd(8) should create home directories for users by default (non
# system users only).
# This option is overridden with the -M or -m flags on the useradd(8)
# command-line.
#
CREATE_HOME     yes
```

3. Verify the user is created.

```
cat /etc/passwd | grep kareem
```

4. Click check to complete the task.
