As part of the temporary assignment to the `Nautilus` project, a developer named `mariyam` requires access for a limited duration. To ensure smooth access management, a temporary user account with an expiry date is needed. Here's what you need to do:  

Create a user named `mariyam` on `App Server 2` in Stratos Datacenter. Set the expiry date to `2026-12-07`, ensuring the user is created in lowercase as per standard protocol.

> Note: You can find the infrastructure details by clicking on the **Details of all Users and Servers** button on the top-right section of the page.

### Answer

1. SSH into the Application Server 2.

```
ssh steve@stapp02
```

2. Check if the user is does not already exist and if not create user with expiration date.

```
# check if user exist
cat /etc/passwd | grep mariyam

# Create user with expiration date
sudo useradd -e 2026-12-07 mariyam
```

3. Check if the user was created and to check the date we can use the `chage` command.

```
sudo chage -l mariyam
```

>**Note**:
>The `chage` command will change the users password expiration information. Using the command without any options will set the days between password expiration for a specific user. 
>
>Using the `-l` however, list all of the users expiration dates, not just password but account and other information. It also provides this in a human readable format. 

4. If the user is created and the expiration date is correct, click check to complete the task.
