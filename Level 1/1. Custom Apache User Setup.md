In response to heightened security concerns, the `xFusionCorp Industries` security team has opted for custom Apache users for their web applications. Each user is tailored specifically for an application, enhancing security measures. Your task is to create a custom Apache user according to the outlined specifications:  

a. Create a user named `anita` on `App server 2` within the Stratos Datacenter.    
b. Assign a unique UID `1896` and designate the home directory as `/var/www/anita`.

> Note: You can find the infrastructure details by clicking on the **Details of all Users and Servers** button on the top-right section of the page.

### Answer

1. SSH into the `App server 2`.

```
ssh steve@stapp02
```

2. Create the user with the following specifications.

```
sudo useradd -u 1896 anita
```

3. Enter the password to create the user.
4. In this case no home directory was created so will use the `usermod` command to specify the home directory. 

```
sudo usermod -m -d /var/www/anita anita
```

5. Check the user and all the specifications are correct by checking the `passwd` file. 

```
cat /etc/passwd | grep anita

# output
anita:x:1896:1896::/var/www/anita:/bin/bash
```

6. Click check to complete the task.

---

The entire process can also be done in one command.

```
sudo useradd -u 1896 -d /var/www/anita anita
```

- The `-u` flag sets the UID.
- The `-d` flag sets a custom home directory.
