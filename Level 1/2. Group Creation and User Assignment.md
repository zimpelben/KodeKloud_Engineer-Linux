The system admin team at `xFusionCorp Industries` has streamlined access management by implementing group-based access control. Here's what you need to do:  

a. Create a group named `nautilus_developers` across all App servers within the `Stratos Datacenter`.  
b. Add the user `stark` into the `nautilus_developers` group on all App servers. If the user doesn't exist, create it as well.

> Note: You can find the infrastructure details by clicking on the **Details of all Users and Servers** button on the top-right section of the page.

### Answer

1. SSH into the Application Server 1.

```
ssh tony@stapp01
```

2. Before creating the group we can check if the user exist. If so we can create the group and add the user in command. 

```
cat /etc/passwd | grep stark
```

3. In this instance the user does not exist so lets create the group and the user and then add the user to the group.

```
# Create group nautilus_developers
sudo groupadd nautilus_developers

# Create user stark
sudo useradd -m stark # -m is to create home dir on creation of user

# Add user to the nautilus_developers group
sudo usermod -G nautilus_developers stark

# Check to make sure stark is part of nautilus_developers group
groups stark
stark : stark nautilus_developers
```

4. The process can be shorted by adding the new user to the group during creation.

```
sudo useradd -m -G nautilus_developers stark
```

- The `-m` flag creates a home directory.
- The `-G` flag adds the user to a secondary group.

4. Exit Server 1 and repeat the steps for Server 2.

```
# Application Server 2

# SSH into server 2
ssh steve@stapp02

# Check if stark user exist
cat /etc/passwd | grep stark # Does not exist on Server 2 in this instance

# Create group nautilus_developers
sudo groupadd nautilus_developers

# Create user stark
sudo useradd -m -G nautilus_developers stark # -m is to create home dir on creation of user

# Check to make sure stark is part of nautilus_developers group
groups stark
stark : stark nautilus_developers
```

5. Exit Server 2 and repeat for Server 3.

```
# Application Server 3

# SSH into server 3
ssh banner@stapp03

# Check if stark user exist
cat /etc/passwd | grep stark # Does not exist on Server 3 in this instance

# Create group nautilus_developers
sudo groupadd nautilus_developers

# Create user stark
sudo useradd -m -G nautilus_developers stark # -m is to create home dir on creation of user

# Check to make sure stark is part of nautilus_developers group
groups stark
stark : stark nautilus_developers
```

6. Exit and click check to complete the task.
